# Лабораторная работа №2: Виртуальный датчик для контроля процесса обжига в печи

## Описание задачи

Разработка виртуального датчика для прогнозирования концентрации целевого продукта в процессе обжига в печи на основе данных телеметрии. Виртуальный датчик позволяет получать оценку качества продукта в реальном времени, не дожидаясь результатов лабораторных замеров (задержка 10-15 минут).

## Описание решения

### 2.1 Разведочный анализ данных (EDA)

- **Телеметрия**: 16 показателей, из которых 12 (telemetry_0–11) имеют >99% заполненность, остальные 4 (telemetry_12–15) удалены из-за >99% пропусков
- **Тренды**: telemetry_0, telemetry_5, telemetry_11 показывают долгосрочные тренды
- **Шум**: telemetry_2, telemetry_7, telemetry_9 имеют высокочастотный шум
- **Выбросы**: telemetry_1, telemetry_6, telemetry_8 содержат резкие аномалии
- **Синхронизация**: оптимальная задержка между телеметрией и лабораторными замерами — **15 минут**

### 2.2 Инжиниринг признаков

Всего создано **156 признаков**:
- **Лаговые признаки**: задержки на 1, 5, 10, 15, 30 минут
- **Скользящие статистики**: mean и std с окнами 5, 10, 30 минут
- **Динамика**: производные (diff) для каждого параметра

### 2.3 Модели

| Модель | Описание |
|--------|----------|
| **Ridge** | Линейная регрессия с L2-регуляризацией, базовая модель |
| **RandomForest** | Ансамбль деревьев, нелинейные зависимости |
| **CatBoost** | Градиентный бустинг, лучшее качество |

**Неподходящие модели:**
- ARIMA/SARIMA — требуют стационарности, не учитывают экзогенные признаки
- OLS — неустойчива к мультиколлинеарности
- KNN — проклятие размерности при 156 признаках
- MLP — избыточна, требует много данных

### 2.4 Результаты

#### Метрики качества (Test)

| Модель | MAE | RMSE | MAPE | WAPE | Direction Acc |
|--------|-----|------|------|------|---------------|
| Ridge | 0.065 | 0.079 | 37.5% | 31.2% | 49.1% |
| RandomForest | 0.063 | 0.077 | 34.5% | 30.0% | 48.1% |
| **CatBoost** | **0.061** | **0.077** | **30.6%** | **28.9%** | **58.8%** |

#### Ключевые выводы

1. **CatBoost — лучшая модель** по всем метрикам
2. **Direction Accuracy 58.8%** — модель улавливает направление изменений лучше случайного
3. **Важнейший признак**: `telemetry_0_diff` — скорость изменения температуры/давления
4. **Автокорреляция остатков** указывает на потенциал улучшения через авторегрессионные компоненты

#### Информационные критерии

| Модель | AIC | BIC | Время предсказания |
|--------|-----|-----|-------------------|
| Ridge | -397 | 190 | ~1.2 мс |
| RandomForest | -412 | 175 | ~15 мс |
| **CatBoost** | **-416** | **171** | ~2.9 мс |

## Интерпретация результатов

### Физические закономерности

1. **telemetry_0** и его производные доминируют — вероятно, ключевой технологический параметр (температура/давление)
2. **Динамические признаки (_diff)** важнее статических — процесс обжига инерционен, важна скорость изменений
3. **Лаги 15-30 минут** согласуются с физической задержкой процесса

### Ограничения

- MAPE ≈ 30% — существенная погрешность, модель склонна к консервативным предсказаниям
- Экстремальные значения (пики/провалы) плохо предсказываются
- Для production требуется дополнительная валидация на реальных данных
