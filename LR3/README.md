# Лабораторная работа №3

## Классификация временных рядов и прогнозирование метеоданных

Вся работа выполнена в ноутбуке `lr.ipynb`.
Реализован двухэтапный пайплайн:
1. Классификация города по временному ряду.
2. Специализированный прогноз температуры на 30 дней для выбранного города.

## Что есть в проекте

- `lr.ipynb` - основной ноутбук.
- `requirements.txt` - зависимости.
- `Лабораторная работа №3.md` - текст карточки задания.

В `data/` положите исходные parquet-файлы перед запуском ноутбука.

## Какие данные нужны

Используемые колонки:
- `temperature_2m`
- `relative_humidity_2m`
- `precipitation`
- `rain`
- `snowfall`
- `weathercode`
- `wind_speed_10m`
- `surface_pressure`

## Проделанная работа

### 2.1 EDA
- Загрузка parquet по всем городам.
- Приведение к дневной частоте и интерполяция пропусков по времени.
- Графики рядов, распределения, boxplot, Q-Q plot, корреляции.
- Проверка стационарности (ADF).

### 2.2 Признаки
- Календарные и циклические признаки.
- Лаги, rolling mean/std.
- Производные температуры и динамика давления.
- Оконные агрегаты и PCA-визуализация.

### 2.3 Классификация
- В ноутбуке присутствуют:
  - baseline на `ExtraTreesClassifier`;
  - специализированный TS-классификатор `MiniRocketClassifier` (из `aeon`).
- Метрики считаются на тестовом периоде 2025 года.

### 2.4 Оценка классификации
- `Accuracy`, `Precision`, `Recall`, `F1` (macro/weighted), `ROC-AUC OvR`.
- Confusion matrix.
- Анализ по сезонам и по месяцам.
- Permutation importance через surrogate RandomForest на оконных фичах.

### 2.5 Прогнозирование
- Отдельные модели по городам.
- Кандидаты: `SARIMAX_EXOG`, `ETS`, `AutoReg`.
- Split: train (2019-2023), val (2024), test (2025).
- Выбор лучшей модели по MAE на validation.

### 2.6 Оценка прогноза
- `MAE`, `RMSE`, `MAPE`, `WAPE`, `DirectionAccuracy`.
- Тесты остатков: Шапиро-Уилка, Льюнг-Бокса, Уайта.
- Графики факт/прогноз, распределение ошибок, прогноз на +30 дней.

### 2.7 Интеграция
- Двухэтапный пайплайн:
  - определение города по 30 последним значениям температуры (через DTW в интеграционной функции);
  - прогноз выбранной специализированной моделью.

### 2.8 Интерпретация
- Формируется итоговая `summary_table` с метриками классификации/прогноза и диагностикой.
- В конце ноутбука дан текстовый вывод по качеству и ограничениям.

## Полученные результаты

- Классификация: `accuracy = 0.8476`, `f1_macro = 0.8416`, `ROC-AUC OvR = 0.9086`.
- Прогнозирование (по городам):
  - лучший MAE: `1.4207` (Геленджик, `SARIMAX_EXOG`);
  - худший MAE: `4.0664` (Находка, `AutoReg`);
  - диапазон RMSE: от `1.8170` до `5.0652`;
  - DirectionAccuracy: от `0.5357` до `0.6456`.
- Выбор моделей по городам: в 5 из 6 городов лучшей стала `SARIMAX_EXOG`, для Находки - `AutoReg`.

  ## Интерпретация результатов

  - Двухэтапный подход работает корректно: классификация достаточно устойчива (`accuracy` около 0.85), а затем специализированные модели дают прогноз разумного качества для большинства городов.
  - Наиболее стабильные результаты получены для городов с более регулярной динамикой температуры (минимальные ошибки MAE/RMSE), что подтверждает адекватность выбранной схемы «классификация -> специализированный прогноз».
  - Находка остается самым сложным кейсом (наибольший MAE), что согласуется с более сложной структурой ряда и указывает на необходимость дополнительной настройки модели именно для этого города.
  - Значения DirectionAccuracy выше 0.53 показывают, что модели в целом улавливают направление изменения температуры, но не всегда точно восстанавливают амплитуду колебаний.
  - Практический вывод: текущий пайплайн подходит как рабочий базовый прототип, а дальнейшее улучшение стоит сосредоточить на городах с максимальной ошибкой и на усилении сезонной компоненты в прогнозных моделях.
